yok3x 사용 설명서

Claude·Codex·Gemini CLI를 묶어 구독 한도를 지키며 코딩 루프(구현→리뷰→수정)를 돌리는 도구. 한 모델이 만들고 다른 모델이 리뷰·채점한다. 의존성 0.

1. 빠른 시작
python yok3x.py init          # 설정·디렉터리 생성 (드라이런은 init --mock)
python yok3x.py setup         # 샘플 태스크 3종 생성
python yok3x.py gui           # 이 브라우저 콘솔 실행
python yok3x.py run task-producer-reviewer.json --auto

--mock 은 실제 CLI 없이 전체 흐름을 검증하는 드라이런이다.

2. 명령어
명령기능
init [--mock]yok3x.json/backends.json/디렉터리 생성
setup샘플 태스크 3종 + 초기화
run <task> [--auto]태스크 1회 실행. --auto 없으면 스텝마다 승인 게이트
loop <task> -n N에이전트 루프. 가드가 stop이면 스스로 정지
gui [--port N]브라우저 콘솔(실데이터)
mat사용량·코칭·최근 런 한 화면(터미널)
coach / coach guard on|off사용량 코칭 / 가드 토글
limits실제 한도 probe 원본 확인(진단)
plan [tool] [name]요금제 프리셋(claude: pro/max5x/max20x)
calibrate claude 5h|7d <%>실사용 기반 claude 한도 정확 보정
knot save|query|lint지식그물(공유 기억)
flavor [이름]오케스트레이터 전환
3. 워크플로우 패턴 3종
  • producer-reviewer — 한 워커가 만들고 다른 워커가 SCORE로 채점, 기준 미달 시 재작업. (구현↔코드리뷰)
  • pipeline — 이전 단계 출력이 다음 입력. (설계→구현→리뷰)
  • fanout-fanin — 여러 워커가 각자 제안 → 하나로 통합.
4. 구독 한도 & 가드 — 한도는 무조건 지킨다

매 워커 호출 전 한도를 점검하고, hard_ratio(기본 100%) 도달 시 루프가 스스로 멈춘다. 소스 배지:

배지도구의미
실측codexapp-server가 지금 값을 직접 보고(가장 정확)
추정claudetranscript 롤링 합산 vs 설정 상한(근사)
원장gemini자체 일일 예산(라이브 소스 없음)
claude는 요금제를 넣거나(plan claude max20x), 앱 /usage의 실제 %를 calibrate claude 7d 30으로 넣으면 정확해진다. codex는 자동.
5. 코딩 태스크 옵션

task.json에 아래를 넣으면 실제 코드를 짜고 돌려서 검증한다:

{ "workdir": "F:/my/repo",
  "verify_cmd": "pytest -q",      // 테스트/린트 게이트: 실패면 높은 SCORE여도 통과 불가
  "verify_timeout_sec": 300,
  "context_globs": ["src/**/*.py"],// 레포 파일을 프롬프트에 주입
  "rubric": "rubric.md" }          // 채점표를 검수에 주입

스톨 감지: 점수·이슈가 2회 연속 같으면 수렴 실패로 조기 종료.

6. 워커 · 안전장치
  • 워커: claude-main(구현) · codex-main(리팩터) · codex-critic(코드리뷰) · gemini(설계검토). 각 워커는 CLI(backend)에 바인딩.
  • 승인 게이트: 스텝마다 y승인/N건너뜀/q중단. --auto는 신뢰된 루프에서만.
  • yok3x 기법: 코딩 워커에 계획→구현→자가검증 구조 주입.
  • 할루시네이션 방지: 날조 금지·근거 제시 지침 + 검수 시 환각 지적 + 과잉확신 자동 표시.
  • 폭주 방지 4중 브레이크: 단계분할 · 승인 · 검증 · 예산(가드).
7. 이 GUI 사용법
  • 대시보드: 도구별 5h/7d 한도·코치·최근 런(실데이터, 7초 갱신).
  • 콘솔: 작업을 입력해 실행(채팅형) + CLI-역할 배치 + 실시간 사용량.
  • 설정·한도: 가드 임계·요금제·calibrate.
  • knot: 공유 기억 검색/저장/lint.
