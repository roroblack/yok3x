# v4.0.0 · 계획서 — 조건부 라우팅(에스컬레이션) · 2026-07-15

## 목표 (LangGraph 이식 아이디어 ①)
지금 패턴은 정적(고정 순서). LangGraph의 **조건부 엣지**를 가져와, **출력/점수에 따라 다음 단계를 동적
으로** 결정한다. 1차 구현: producer-reviewer에서 **수렴 실패 시 에스컬레이션** — "N라운드 후 점수가
낮으면 더 강한 모델/다른 워커로 전환". 이미 있는 스톨감지·적응형 열화의 자연스러운 일반화.

## 현재 구조(근거)
- `run_producer_reviewer` 루프: 매 라운드 producer→reviewer, `score`·`verify_ok`로 통과 판정, 스톨감지로
  조기종료. producer/reviewer는 **런 내내 고정**.

## 설계 (의존성0, 최소)
- task spec에 선택적 **`escalate`** 규칙:
  ```json
  "escalate": {"after_round": 2, "if_score_below": 6, "to_producer": "codex-main", "to_reviewer": "gemini"}
  ```
  (필드 일부만 지정 가능. to_* 없으면 해당 역할 유지.)
- `run_producer_reviewer`: 라운드가 **통과도 스톨도 아닐 때**, `rnd >= after_round`이고
  `score < if_score_below`이며 아직 에스컬레이션 안 했으면 → 이후 라운드의 producer/reviewer를
  `to_producer`/`to_reviewer`로 **1회 전환**(로그 `[escalate] round N: reviewer X→Y`). 교차패밀리 유지.
- 안전: 전환 대상 워커가 없으면 무시(경고). 1회만(스래싱 방지). 기존 스톨 조기종료·통과 판정은 그대로.

## 확장 여지(후속)
- 일반 조건부 전이(spec `on: [{when:..., goto:...}]`)로 임의 DAG — 지금은 범위 밖(과설계 지양).
- verify 실패 유형별 라우팅(보안 결함 → 보안 워커) 등.

## codex 리뷰 반영 (SCORE 7/10)
- **다음 라운드부터 적용**(같은 라운드 재실행 금지), **정확히 1회**(escalated 플래그) — 반영.
- **대상 사전검증**: to_producer/to_reviewer가 없는 워커면 조용한 폴백 대신 **RunAborted로 실패** — 반영.
- **prev_sig 초기화**: 워커 전환 시 스톨 시그니처 리셋(새 워커 출력이 옛 시그니처와 오탐 비교 방지) — 반영.
- 에스컬레이션은 통과 판정 뒤·스톨 판정 앞에서 트리거해 스톨로 조기종료되기 전 새 워커에 기회 부여.
