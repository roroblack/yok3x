# BUG-02 · 버전 표기 드리프트

- **시점**: 2026-07-10 · **심각도**: 낮음 · **커밋**: `39dff80` · **상태**: 수정됨 ✅

## 증상
`__version__ = "3.1.0"`인데 사용자가 보는 화면은 옛 버전:
- `matview.py` mat 배너: `v2.2`
- `cli.py` 설명: `v2.2`
- `__init__.py` 독스트링: `v3.0`

RULE §7(이력 정직성)인데 정작 화면 버전이 3곳에서 3개로 갈라져 있었다.

## 근본 원인
버전 문자열이 여러 파일에 **하드코딩**돼 있어, `__version__`만 올리고 표시 문자열은 안 따라감.

## 수정
표시 버전을 `__version__` **단일 출처**에서 f-string으로 주입(mat 배너·cli 설명). 독스트링은
리터럴 정정. 이후 `_version.py`로 버전을 분리해(BUG 아님, 패키징 편의) 단일 출처 강화.

## 검증
`mat`·`cli -h` 모두 `v3.3.0` 표시 확인. `yok3x/*.py`에 옛 버전 리터럴 잔존 0.
단위 테스트 `test_version_is_single_source`(배너에 `__version__` 포함, `v2.2`/`v3.0` 부재).
codex로 보내는 `limits.py` clientInfo `"2.2"`는 무해하여 의도적으로 유지(정직 표기).

## 교훈
사용자에게 보이는 버전은 **반드시 단일 출처**에서. 하드코딩 복제는 조용히 드리프트한다.
