# BUG-03 · 스톨 감지 신호 약함 + 사(死)변수

- **시점**: 2026-07-10 · **심각도**: 중간 · **커밋**: `39dff80` · **상태**: 수정됨 ✅

## 증상
producer-reviewer 루프의 스톨(수렴 실패) 조기 종료가 의도대로 동작하지 않았다. "같은 이슈
반복"을 잡으려 했으나 사실상 "점수만 반복"으로 축약됐고, `stall` 카운터는 무의미했다.

## 근본 원인
`run_producer_reviewer`에서 스톨 판정에 쓰던 `issues_sig = self.steps[-1].checklist`가
리뷰어 **응답의 메타 품질**(`_checklist`: 빈 응답·과대·거부·과잉확신)이지, **리뷰어가 산출물에
대해 지적한 실제 결함**이 아니었다. 정상 응답이면 거의 항상 `()` → `sig=(score,())`로 축약.
또 `if stall >= 1`이 첫 중복에 바로 break라 `stall` 누적이 의미 없는 사변수였다.

## 수정
- `_defect_sig(text)` 신설: 리뷰어 텍스트에서 SCORE 줄·글머리표·번호·구두점을 제거·정규화한
  **지적 결함 집합**을 서명으로. 순서·번호·가벼운 재서술에 견딤.
- 스톨 판정을 `(score, _defect_sig(rev.text))`로. 사변수 `stall` 제거, "직전 라운드 동일 시 종료"로 정직화.

## 검증
단위 테스트: `_defect_sig` 순서/번호 무시 동일, 다른 결함 구분, 빈/점수만 응답 → `()`.

## 교훈
"반복 감지"는 **무엇을 서명으로 삼는가**가 전부다. 응답의 메타 품질이 아니라 의미 있는 내용을 해시한다.
