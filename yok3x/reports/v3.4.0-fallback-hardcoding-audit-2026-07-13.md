# v3.4.0 · 폴백·하드코딩 감사 레지스트리 · 2026-07-13

RULE §5.5가 참조하는 **현행 폴백/하드코딩 추적 레지스트리**(그동안 부재 → 신설). 새 폴백/하드코딩을
추가하면 여기 등록한다. 각 항목: 위치 · 왜 필요 · 실패 시 표시 · 설정으로 뺄 수 없는 이유.

허용 기준(§5.5): 폴백은 **명시적 열화가 사용자에게 표시**될 때만. 하드코딩은 **프로토콜 상수·안전
상한**만(사용자 환경마다 다른 값은 설정으로).

---

## A. 사용량 실측 폴백 체인 (limits.py) — 모두 '명시적 열화'

| # | 위치 | 폴백 | 표시 | 판정 |
|---|---|---|---|---|
| A1 | `_probe_claude_oauth` | live(OAuth) 실패 → **최근 실측 stale 유지**(≤`max_stale_sec` 900s) | `⚠N분 전 실측·사유` | ✅ 명시적 |
| A2 | 〃 | stale도 만료 → **트랜스크립트 추정**(plan/cap 있을 때만) | `추정` 배지(노랑) | ✅ 명시적 |
| A3 | 〃 | 추정 ratio>200%(미보정) → **버리고 원장** | `원장` 배지 | ✅ 명시적 |
| A4 | `check_backend` | 추정 real=False+ratio>3.0 → 정지 유보 **warn**(false-stop 방지) | `warn`+calibrate 힌트 | ✅ 명시적 |
| A5 | `_probe_codex_appserver` | app-server live 실패 → **세션 파일(stale)** | `⚠stale(파일)` | ✅ 명시적 |
| A6 | 〃 | 파일도 실패 → **원장** | `원장` 배지 | ✅ 명시적 |
| A7 | `_probe_uncached` | probe 미설정/비활성 → **원장(일일 예산)** | `원장` 배지 | ✅ 명시적 |
| A8 | `failover_backend`(P3) | 클라우드 전부 stop + 로컬 서버 도달 → **로컬 강등** | `[failover]→local` 로그 | ✅ 명시적 |

**최근 추가(이번 세션):** A1(stale-while-error, `1be4006`) · A3(1003% 차단, `5811eb3`) ·
A4(false-stop 방지, `511fbeb`). 셋 다 배지/detail로 열화를 표시 → §5.5 충족.

## B. 모델 목록 조회 폴백 (limits.py `_fetch_models`)

| # | 위치 | 폴백 | 표시 | 판정 |
|---|---|---|---|---|
| B1 | gemini 분기 | 키 있으면 실시간 API → **없으면 CLI 번들 GEMINI_MODELS 레지스트리** | 드롭다운(번들 소스) | ✅ 실제 소스 |
| B2 | gemini 분기 | 번들도 없음(CLI 미설치) → **빈 목록** | GUI "➕ 커스텀 입력" | ✅ 명시적 |
| B3 | claude/codex | 소스(토큰/캐시) 없음 → **빈 목록** | 드롭다운 비고 | ✅ 명시적 |

gemini 인증은 이 계정에서 **Antigravity/CloudSDK 암호화 OAuth**라 평문 키·토큰이 없어 yok3x가 못
읽는다(gemini CLI 자체는 인증됨). 그래서 키 없을 때는 **gemini CLI 번들의 `GEMINI_MODELS` Set**
(설치된 CLI 버전이 지원하는 실제 모델 집합, `_gemini_bundle_models`)을 읽는다 — codex의
`models_cache.json`과 동급의 실제 소스로 **하드코딩 아님**(CLI 업데이트 시 자동 갱신). 취약점: 번들이
미니파이 JS라 파서가 `var X="gemini-.."` + `GEMINI_MODELS Set` 패턴에 의존 → CLI가 구조를 크게 바꾸면
빈 목록으로 열화(명시적). API 키(`api_key`/`api_key_path`/`api_key_env`)를 주면 실시간 API가 우선.

## C. 허용 하드코딩 (프로토콜 상수·안전 상한)

| # | 값 | 성격 | 판정 |
|---|---|---|---|
| C1 | 창 크기 300분→"5h", 10080분→"7d" (`_window_name`) | 프로토콜 상수 | ✅ 허용 |
| C2 | plan 프리셋 pro/max5x/max20x 토큰 상한 | 추정용 근사 상한(설정으로 override 가능: `limit_5h_tokens`) | ✅ 허용 |
| C3 | timeout(15s 등)·`max_stale_sec`·`min_interval_sec` | 안전 상한(설정 가능) | ✅ 허용 |
| C4 | API 필드명·엔드포인트 URL(anthropic/google) | 프로토콜 상수(URL은 설정 override 가능) | ✅ 허용 |
| C5 | 버전 문자열 | `_version.__version__` 단일 출처 | ✅ (하드코딩 제거됨) |

## D. 위반/부채 — 없음(현재)

- 조용히 값을 지어내는 폴백 없음. 사용자 환경별 값(예산·한도·경로·모델명·키)은 전부 `yok3x.json`으로 분리됨.

---

**요약**: 현행 폴백은 전부 배지/detail로 열화를 **표시**하는 명시적 폴백이고, 하드코딩은 프로토콜
상수·설정가능 안전 상한뿐. §5.5 기준 충족. 이 파일이 이후 추적의 기준 레지스트리.
