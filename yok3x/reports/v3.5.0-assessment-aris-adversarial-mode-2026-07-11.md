# v3.5.0 평가/계획 — ARIS(적대적 멀티에이전트) yok3x 적용

작성: 2026-07-11 · 출처: ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
(arXiv:2605.03042) · **결론: 적용 가치 매우 높음. yok3x는 이미 70~80% 구현 상태 — "적대적 모드"로 마무리하면 됨.**

---

## 1. ARIS 핵심 (요약)

장시간 LLM 에이전트가 **"그럴듯하지만 근거 없는 성공"**(plausible unsupported success)을 내는 것을
막는 자율연구 하네스. 핵심 장치:
- **교차모델 적대적 협업(기본값)**: executor가 연구를 추진, **다른 모델 패밀리의 reviewer**가
  중간 산출물을 비판·수정요구 → 근거 없는 주장이 전파되기 전에 표면화.
- **3층**: 실행(스킬·MCP·연구 위키) / 오케스트레이션(워크플로 + reviewer 라우팅 설정) /
  **보증**(무결성·result↔claim 매핑·감사, 다중 패스 편집).

## 2. yok3x는 이미 얼마나 가까운가 (매핑)

| ARIS 요소 | yok3x 현황 |
|---|---|
| executor + reviewer 교차검증 | ✅ **producer-reviewer**(한 모델 생성 → 다른 모델 채점) |
| **다른 모델 패밀리** reviewer | ◑ routing이 claude 생성→codex/gemini 검수로 보내나 **강제 안 함** |
| 근거 없는 주장 차단 | ◑ `anti_hallucination`(과잉확신 플래그) + `REVIEW_GUARD` + verify_cmd 게이트 |
| result↔claim 매핑·감사 | ✗ 없음(주장-증거 대응 감사 단계 부재) |
| 지속 연구 위키 | ✅ `knot`(md 지식그물·[[위키링크]]) |
| reviewer 라우팅 설정 | ✅ flavors/routing + v3.3 상황별 프로파일 |

→ **핵심 패턴(교차모델 producer-reviewer)은 이미 있다.** ARIS와의 격차는 "채점"을 넘어선
**적대성 강화**와 **주장-증거 감사**다.

## 3. "적대적 모드"로 추가할 것

### A1 — 적대적(red-team) 리뷰어 프롬프트
현재 리뷰어는 "SCORE + 결함 목록". 적대적 모드에선 **"이 산출물을 반증/파괴하라"**:
가장 강한 반례·미검증 가정·엣지케이스 실패를 적극적으로 찾고, "통과시키지 못하는 이유"를
우선 서술. (기존 `REVIEW_GUARD`에 red-team 지침 추가 + 새 프롬프트 상수)

### A2 — 교차 패밀리 강제
producer와 reviewer가 **같은 모델 패밀리면 경고/자동 교체**(claude↔codex↔gemini). ARIS의
"different model family" 기본값을 옵션(`require_cross_family`)으로. v3.3 `resolve_model`·
프로파일과 결합하면 상황별로 서로 다른 패밀리 자동 배정 쉬움.

### A3 — 주장↔증거 감사 단계(assurance)
producer 산출의 각 **주장을 추출**하고 근거(파일·실행결과·인용)를 대응시켜, 근거 없는 주장을
표시하는 **감사 스텝**을 verify 게이트 옆에 추가. 근거 미매핑 주장은 SCORE 상한을 낮춤.
(anti-hallucination의 강화판, 객관 게이트화)

### A4 — 새 오케스트레이션 패턴 `adversarial`(토론/디베이트)
단발 producer-reviewer를 넘어 **다라운드 적대 토론**: A 제안 → B 공격 → A 방어 → 판정자
종합. 기존 패턴(producer-reviewer/pipeline/fanout) 옆에 `adversarial` 추가. 스톨 감지·가드·
프로파일 그대로 재사용.

## 4. 내 평가 (권고)

**하자 — 적합도 최상, 착수 비용 낮음.** yok3x의 철학(교차검증·할루시네이션 방지·검증 게이트·
정직성 §7)과 ARIS는 사실상 같은 지향이다. 이미 깔린 producer-reviewer·anti-hallucination·
verify·knot·프로파일 배관을 재사용하므로 새 의존성 0으로 구현 가능.

**원칙**: (1) opt-in 모드(기본 동작 불변), (2) 적대성은 **품질 게이트를 강화**하되 리뷰어를
약화(다운그레이드)시키지 않음(P1 `roles_no_downgrade`와 정합), (3) 모든 판정 근거 명시 로깅.

## 5. 단계

| 단계 | 범위 | 규모 |
|---|---|---|
| **AD1** | 적대적 리뷰어 프롬프트(A1) + 교차패밀리 강제 옵션(A2). task/프로파일에서 켜기 | 반나절 |
| **AD2** | 주장↔증거 감사 스텝(A3) — verify 옆 객관 게이트, 근거 미매핑 시 SCORE 상한 | 반나절~1일 |
| **AD3** | `adversarial` 디베이트 패턴(A4) + GUI 패턴 선택지·근거 표시 | 1일 |

**한 줄**: yok3x = 코딩 특화 producer-reviewer, ARIS = 연구 특화 적대적 검증. 같은 뼈대이므로
"적대적 모드"는 자연스러운 상위 기능. AD1부터 최소 구현 권장.
