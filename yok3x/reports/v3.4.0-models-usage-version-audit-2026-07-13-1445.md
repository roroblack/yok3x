## 2. 코덱스 "5h가 사라지고 7d만" = 회귀 아님 (원본 데이터가 7d 하나뿐)

codex app-server가 반환한 **원시 rate_limits**(롤아웃 캡처):
```json
{"primary": {"used_percent": 2.0, "window_minutes": 10080, "resets_at": ...},
 "secondary": null, "plan_type": "plus"}
```
- **primary = window_minutes 10080 = 7일**, secondary = **null**. 즉 codex(plus)는 지금 **7d 창 하나만** 보고한다.
- 예전 "5h" 표시는 이 7d 창을 **위치(primary)로 잘못 라벨**한 것(BUG-12). 길이(windowDurationMins) 기준으로
  고쳐 "7d"로 **정확**해진 것 — 5h를 없앤 게 아니라 **원래 5h가 안 온다**. 조치 불필요(정상).

## 3. 제미나이 모델 목록 = 실제 Google API로 구현 (키 있으면 실시간)

- **진짜 동적 소스**: `GET https://generativelanguage.googleapis.com/v1beta/models?key=KEY`
  (gemini SDK/CLI가 쓰는 그 목록). `supportedGenerationMethods`에 `generateContent` 있는 모델만 반환.
- **키 해석(config 주도, `limits.gemini`)**: `api_key` → `api_key_path` 파일 → `api_key_env`
  (기본 `GEMINI_API_KEY`/`GOOGLE_API_KEY`/`GOOGLE_GENAI_API_KEY`). 키는 커밋 금지.
- **이 PC에서 목록이 비는 이유**(명시적 폴백, §5.5): gemini CLI는 API 키를 **디스크에 평문 저장하지
  않고 실행 시 env로 읽는다**. 조사 결과 env(Process/User/Machine)·자격증명관리자·`~/.gemini` 어디에도
  평문 키가 없었다(자격 탐색은 분류기 정책상 중단). → 키 없으면 `[]` 반환 + GUI는 "커스텀 입력" 안내.
- **완성 방법(사용자)**: `GEMINI_API_KEY` 환경변수를 설정하거나 `limits.gemini.api_key_path`에 키 파일
  경로를 넣으면, 재조회 시 드롭다운이 **실시간 gemini 모델 목록**으로 채워진다. 코드 변경 불필요.
- 폴백 성격: 조용히 가짜 목록을 만들지 않고 **빈 목록 + 안내**(명시적 열화). §5.5 허용 기준 충족.

## 4. 버전 "3.1" 오표시 → 단일 출처(3.4.0)로 정정 (데이터 정정, §5.6 예외)

- 원인: `config.py`가 `"version": "3.1"`을 **하드코딩**해 `_version.__version__="3.4.0"`와 불일치.
  GUI 헤더는 `state.version`을 그대로 표시 → 3.1로 보임.
- 조치: `config.py`·`guiserver.build_state`가 **`_version.__version__` 단일 출처**를 쓰게 변경. 저장된
  `yok3x.json`의 낡은 version에도 오염되지 않도록 build_state는 코드 버전을 직접 보고. GUI 문구/레이아웃
  변경 없음(데이터 값만 정정) → §5.6 예외에 해당.

## 5. RULE §5.6 신설 — GUI/UX는 사용자 명령 없이 수정 금지

- 사용자 요청으로 `RULE.md`에 **§5.6**을 추가: 명령 없이 `gui/`·레이아웃·문구·색·상호작용을 바꾸지
  않는다. 바꿔야 하면 **리포트로 제안 후 승인**. 예외는 백엔드가 넘기는 데이터 값 정정뿐.
- 이번 변경 중 GUI 파일(`gui/index.html`)은 **건드리지 않았다**(모델 목록·버전 모두 백엔드 데이터로 처리).
