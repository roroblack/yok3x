# v3.5.0 · knot consolidation(Mem0식, 의존성 0) · 2026-07-13 16:00

Mem0 평가([v3.5.0-assessment-mem0](v3.5.0-assessment-mem0-2026-07-13.md)) 권고 이행: 벡터DB·추가
LLM 없이 knot에 **추출·통합(consolidation)** 3종을 의존성 0으로 이식. LightRAG 이중레벨 검색과 보완.

## 구현 3종

1. **최신성 감쇠(recency decay)** — `_recency_weight(created, now, halflife)` × 검색 점수.
   반감기(`knot.recency_halflife_days`, 기본 90일)마다 점수 절반 → **낡은 기억 자연 강등**.
   0이면 끔. created 파싱 실패/나이 0이면 1.0(무해). `query`에 통합.
2. **중복 통합(dedup)** — `lint`에 노트 쌍 유사도(`_similarity`: 태그·링크·제목토큰 자카드 평균)가
   `knot.dedup_threshold`(기본 0.6) 이상이면 **"중복 후보 — 통합 검토"** 표시. 임베딩 없이 그래프/텍스트.
3. **요점 저장(key-point extraction)** — `extract_key_points`: 원문 전체 대신 SELF-CHECK·SCORE·결정·
   결함 등 **결론 신호 줄만** 응축. 오케스트레이터 `_finish`가 런을 knot에 저장할 때 이걸 사용 →
   지식그물 비대·중복 억제. **새 LLM 호출 없음**(워커가 이미 낸 구조 재사용).

## 원칙
- 의존성 0(표준 라이브러리만). 벡터DB·임베딩·추가 LLM 없음 → Mem0 프레임워크의 과설계 회피.
- 설정으로 조절(halflife·threshold). 기본값은 기존 동작을 크게 바꾸지 않게 온건(90일·0.6).

## 검증
- 신규 테스트 5: recency 가중 공식·최신 우선 랭킹·중복 감지(유사쌍만)·요점 추출. knot 그린.
- 기존 이중레벨 검색 테스트 유지(최신성은 created가 비슷한 노트엔 영향 미미).

## 남은 것(범위 밖)
- 자동 병합(현재는 감지·표시까지). 사용자/후속 런이 판단해 병합.
- 임베딩 기반 의미 유사도가 정말 필요하면 외부 backend(코어 밖) 선택 옵션.
