# v3.5.0 · 계획서 — P3 오프라인 폴백(로컬 모델) · 2026-07-13 15:20

## 목표
클라우드 백엔드(claude/codex/gemini)가 **전부 한도/레이트리밋(429 등)에 걸리면 멈추는 대신**,
로컬 모델로 자동 강등해 **무중단**으로 계속 돌린다. 기존 적응형 열화(P1 모델다운·P2 백엔드폴오버)의
자연스러운 **3단계(P3)**. 이번 세션의 429 폭풍이 직접 동기.

## 원칙 준수
- **의존성 0 유지**: 로컬 런타임(Ollama 등)은 `localhost:11434` HTTP를 `urllib`로 호출(표준 라이브러리).
  claude/codex/gemini CLI를 깐 것과 동급 — 사용자가 런타임만 설치하면 됨. yok3x 코어에 새 패키지 0.
- **§5.5**: P3는 명시적 열화(배지/로그에 `⚠오프라인(local)` 표시). 레지스트리 등록.
- **§5.6**: GUI 레이아웃 변경 없음. backend_models/배지는 기존 데이터 경로 재사용.

## 사전 발견 (중요)
- 이 PC엔 **로컬 런타임 미설치**(ollama/llamafile/lm-studio 없음, `gemini gemma` 바이너리도 미설치).
- → **기계(어댑터+P3 티어+테스트)는 지금 만들고 목(mock) HTTP로 단위검증**. **실동작 데모는 사용자가
  런타임 설치 후**(예: `winget install Ollama.Ollama` + `ollama pull qwen2.5-coder:7b`). 설치는 다운로드·
  실행이라 사용자 승인 영역 — 제가 대신 안 함.

## 체크리스트 (단계별)

- [ ] **P1. 로컬 HTTP 백엔드 어댑터**
  - `backends.json`에 `local`(type: http) 추가: Ollama `/api/chat`(또는 `/api/generate`) 호출.
  - `call_worker`가 CLI(subprocess) 외에 **http 백엔드**도 실행하도록 분기(urllib POST, stdin=프롬프트).
  - `config.py`: `workers`에 `local-main`(backend: local) + flavor에 옵션 포함(주입만, 기본 미사용).
- [ ] **P2. 로컬 모델 목록** (`list_models("local")`)
  - Ollama `/api/tags` → 설치된 모델 슬러그. 기존 backend_models 경로에 자동 표시.
- [ ] **P3. 강등 티어(핵심)**
  - `usage`: 모든 클라우드 백엔드가 guard `stop`이면 → `failover_backend`가 **local**을 마지막 후보로 반환.
  - `limits.gemini`/config에 `offline_backend: "local"`·`offline_enabled`(기본 on) 설정.
  - `orchestrator.call_worker`: P2 폴오버 소진 후 P3로 local 전환, 로그 `[P3] 오프라인 강등(local/<model>)`.
  - 리뷰어(품질 게이트)는 P3 제외(`roles_no_failover`처럼) — 로컬은 생산자/글루에만.
- [ ] **P4. 테스트(목)**
  - http 어댑터: urlopen 목으로 요청/응답 파싱.
  - list_models(local): /api/tags 목.
  - P3 강등: 클라우드 전부 stop → failover가 local 반환. 리뷰어는 강등 안 됨.
- [ ] **P5. 문서·감사**
  - HISTORY·§5.5 레지스트리(P3 오프라인 폴백 등록)·config 주석. 실동작 데모는 런타임 설치 후로 명시.

## 미적용(범위 밖, 후속)
- 로컬을 **주 코드 생산자/리뷰어**로 두는 flavor(품질 저하) — 안 만듦.
- Mem0식 knot consolidation을 로컬로 돌리는 것은 별도 계획.

- 49+신규 테스트 그린. GUI 미변경. 런타임 설치 시 실동작 가능(문서화).
