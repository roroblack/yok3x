# v3.7.0 · 계획서 — 작업(task)별 콘솔 뷰 · 2026-07-14

## 목표
GUI '작업 콘솔'을 **작업별로 그룹핑**해서 본다. 공통사항(전체 상태·사용량·가드)은 따로 빼고, 각
작업(task)의 런 이력·스텝 로그를 그 작업 그룹 안에서 본다. 현재는 최근 런 N개를 **평탄 리스트**로만 표시.

## 현재 구조(근거)
- `guiserver._recent_runs`가 런 폴더(`.yok3x/runs/*/status.json`)에서 최근 N개를 읽어
  `{run_id, pattern, task(자유문자열), state, steps, stepdetail[worker/kind/status/score/summary/issues]}` 반환.
- GUI `renderConsole`가 이걸 카드로 나열. 공통 정보(사용량 미터·코치·가드)는 다른 패널에 이미 있음.

## codex 논의 결과(반영)
- **그룹키 = 불변 식별자 + 표시용 `label` 조합**. label 미지정 시 **task 파일명** fallback.
- **task 자유문자열로 그룹핑 금지** — 문구 변경·충돌에 취약. 텍스트는 검색/유사작업 추천에만.
- **UI = 필터 드롭다운(확장성)** + 선택된 작업의 로그를 아코디언으로 접는 **혼합형**.
- 공통 영역: 전체 상태·진행률·실행시간·공용 환경/브랜치·전역 경고. task별: 로그·산출물은 그룹 안.

## 설계

### 1) 그룹키 — `label` 도입(불변키 아님, 표시·그룹용)
- 런 실행 시 선택적 **`label`**(예: "슬러그 함수", "LRU 캐시")을 받아 `status.json`에 저장.
  - 소스: task.json의 `label` 필드 / GUI 런 입력창의 라벨 / 없으면 **task 파일명**(task-producer-reviewer)
    또는 최종 fallback **"무제목"**.
- 그룹키 우선순위: `label` → task 파일 basename → `"무제목"`. **자유 task 텍스트는 키로 안 씀**.
- 불변 `run_id`는 각 런 식별용(그룹 내 정렬·상세).

### 2) 백엔드(의존성0, `_recent_runs` 재사용)
- `orchestrator`가 런 시작 시 spec의 `label`을 `status.json`에 기록(없으면 task 파일명/무제목).
- `_recent_runs`에 `label` 필드 추가(기존 필드 유지). `build_state`에 **`run_groups`**(label→런목록) 또는
  런 배열에 label만 얹고 GUI에서 그룹핑(택1 — GUI 그룹핑이 단순).
- N 상한 상향 옵션(작업별 보려면 더 많은 히스토리 필요) — `runs_max` 설정.

### 3) GUI(§5.6 — 별도 승인 후 P구현)
- 콘솔 상단 **필터 드롭다운**: `작업: [전체 ▾ | 라벨1 | 라벨2 | 무제목]`.
- 전체=현행 평탄 리스트. 특정 라벨 선택=그 작업의 런들만, 각 런 **아코디언**(스텝 로그 접기).
- **공통 패널 분리**: 사용량·가드·페이싱·전역 경고는 상단 고정(작업 무관). 작업 그룹은 그 아래.
- 런 실행 입력창에 **라벨 입력**(선택) 추가.

### 4) 엣지케이스·마이그레이션
- 라벨 없는 기존 런: task 파일명 유추 실패 시 "무제목" 그룹.
- 라벨 많음: 드롭다운이 스케일(아코디언 무한 나열 회피 — codex 지적).
- 런 0개/그룹 1개: 드롭다운 숨기고 현행 표시.
- 동일 라벨 다중 런: run_id·시각 내림차순 정렬.
