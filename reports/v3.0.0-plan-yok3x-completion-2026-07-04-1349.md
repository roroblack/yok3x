# v3.0.0 · yok3x 리브랜딩 + 완성 작업 계획서 · 2026-07-04 13:49

멀티 에이전트 코딩 코치 툴을 **`yok3x`**로 리브랜딩하고, 모든 단점을 개선해 버그 없이 동작하는 완성본으로 만든다. 이 문서는 착수 전 계획이며, 완료 후 별도 결과 리포트를 제출한다.

## 0. 안전 원칙 (중요)

- **백업 완료**: `backup/harness-multiagent-v2.3.0-backup-*/` 에 원본 보존. 릴리스 zip(v2.3.0)도 롤백 지점.
- **무인 자기수정 루프는 돌리지 않는다.** 실제 CLI로 프로그램이 스스로를 고치는 루프를 감독 없이 돌리면 (a) 카파시 4원칙이 경고하는 폭주·(b) 요금 소모·(c) 깨진 코드 위험이 있다. 이 툴 자체의 설계 철학에 반한다. 따라서 개선은 직접 수행하고, mock 백엔드 + 실측 probe 경로로 **검증 가능한 것만 반영**한다. 실 CLI 다중 에이전트 실행은 코드 리뷰로 정합성만 확인.
- 매 단계 검증. "될 것"이 아니라 "실행해서 됐다"로만 완료 처리.

## 3. 모든 단점 개선 (코딩 적합성 리포트 기반)

**P1 (본질)**
- **테스트/검증 게이트**: task `verify_cmd` — producer 산출 후 명령(pytest/lint 등) 실제 실행, 결과를 리뷰어에 주입 + 비정상 종료를 **하드 실패**로. 객관 검증 = 할루시네이션 근절.
- **작업 디렉터리**: task `workdir` — 워커/검증을 실제 cwd에서 실행(실 CLI가 파일 편집 가능한 구조).
- **레포 컨텍스트 주입**: task `context_globs` — 관련 파일을 프롬프트에 주입(글자 제한).

**P2 (실용)**
- **스톨 감지**: 점수·이슈가 2회 연속 동일 → 조기 종료 + knot 실패 기록.
- **rubric 주입**: task `rubric` 파일 → 검수 프롬프트에 통째.
- **diff 기반 리뷰 컨텍스트**: 변경분 위주 전달(가능 범위).

**P3 (완성도)**
- **GUI 실행 배선**: `POST /api/run` — GUI에서 태스크 실행, 상태 폴링으로 진행 표시.
- **런 이력/재개 표시**.
