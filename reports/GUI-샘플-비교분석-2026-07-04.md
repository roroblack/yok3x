## 1. 샘플별 요약

### 🟡 A. claude/ — 단일 자체완결 HTML
- **스택**: 순수 HTML+CSS+바닐라 JS, **의존성 0**. 389줄 / 23KB 파일 하나.
- **실행**: 파일 더블클릭. 빌드·설치·서버 불필요. **완전 오프라인**(폰트도 시스템 Comic Sans MS, CDN 0).
- **명세 충실도**: 대시보드(3도구 5h/7d 게이지·소스배지·코치 말풍선·런 타임라인)·태스크 편집기(3패턴)·설정(가드 슬라이더·요금제·**캘리브레이트 필드**)·knot. **소스배지·캘리브레이트·리셋 카운트다운 모두 있음**.
- **디자인**: 코믹 미학을 CSS만으로 구현(지그재그 띠·하프톤·오프셋 그림자·말풍선). 단 손글씨 폰트가 Comic Sans MS라 **레터링 질감은 셋 중 가장 투박**.
- **백엔드**: 없음(정적 목업, 단 값은 실측 반영 — codex 라이브 5h1%/7d24%).
- **장점**: 무설치·오프라인·이식성 최상, 명세 핵심기능 누락 없음, 이해·수정 즉시.
- **단점**: 정적 목업(인터랙션 최소), 단일 파일이라 대형 앱으로 키우려면 재구조화 필요, 폰트 질감 아쉬움.

### 🔵 B. lovable/ — React + TanStack Start (SSR)
- **스택**: React 19 + TanStack Router/Query + Vite 8 + Tailwind v4 + shadcn/Radix. 핵심은 **단일 `index.tsx` 884줄**(정리 잘 됨) + 설정/서버 글루.
- **실행**: `npm install && npm run dev`. **Node + dev 서버 필요**, SSR 앱(`server.ts`/`start.ts`). 무설치 불가.
- **명세 충실도**: **셋 중 가장 완결**. 5개 뷰 전부 + 소스배지(실측/추정/원장) + 리셋 카운트다운 + 소프트 임계 틱 + 코치 로직 실제 계산 + knot 실시간 검색 필터 + 가드 슬라이더 클램프. **단, 캘리브레이트 누락 · stale 배지 없음**.
- **디자인**: **최고 완성도**. Patrick Hand/Fraunces/Nunito 손글씨 폰트, oklch 팔레트, wiggle/bounce 애니메이션, 말풍선, 이모지 브랜딩(🐶🦴). 삽화 에셋은 없음.
- **백엔드**: 없음(전부 하드코딩 목업, 모든 버튼 inert).
- **장점**: 명세 커버리지·인터랙션·디자인 모두 최상, 타입드·가독성 좋은 실제 React 앱.
- **단점**: 100% 목업(배선 0)·**캘리브레이트 누락**, 의존성 ~60개 대부분 미사용(무거움), **폰트 CDN이라 완전 오프라인 아님**, SSR이라 배포 무거움, 단일 파일 모놀리식.

### 🔴 C. harness-gui-snoopy/ — React + Vite (Manus 스캐폴드)
- **스택**: React 19 + Vite 7 + TS + Tailwind v4 + shadcn + wouter. **Manus AI 생성**. 전체 ~8,700 LOC지만 **의미 코드는 ~1,100 LOC**(나머지 shadcn 보일러플레이트 70여 파일).
- **실행**: `pnpm install && pnpm dev`(3000). **프로덕션 `start` 스크립트가 POSIX형이라 Windows에서 실패**(수정 필요).
- **명세 충실도**: 7개 뷰 UI를 폭넓게 스케치. knot **lint는 실제 동작**. 하지만 **소스배지 전무 · 캘리브레이트 없음 · 리셋 카운트다운 없음 · calls/tokens/cost 하드코딩 · 코치 리셋시간이 `Math.random()`**.
- **디자인**: 코믹 테마 CSS 성실 구현(코믹 패널·말풍선·팔레트). 폰트 Courier Prime CDN. 에셋 0.
- **백엔드**: 없음. `server/index.ts`는 24줄 정적 서버뿐.
- **장점**: 뷰 스케치 폭이 넓음, 코믹 CSS 경량, knot lint 실동작.
- **단점**: **명세 핵심(소스배지·캘리브레이트·리셋) 누락**, **라우팅 이원화·죽은 import·딥링크 불가** 등 구조 취약, Manus 잔재(자기복제 zip·미사용 의존성 60+·Windows 비호환 스크립트).

## 4. 목적별 추천 & 다음 단계

**시나리오 1 — 지금 빠르게 쓰는 모니터 패널**
→ **claude 채택**. 남은 일: 작은 로컬 브리지(파이썬 `http.server` + `harness limits/mat --json` 호출 엔드포인트) 붙여 정적 값을 실데이터로 교체. 단일 파일이라 배선 최소.

**시나리오 2 — 제대로 된 웹 대시보드로 발전**
→ **lovable 토대 채택**(디자인·구조 최상). 남은 일: (a) 캘리브레이트 뷰 추가, (b) Google Fonts를 self-host로 바꿔 오프라인화, (c) 미사용 의존성 정리, (d) SSR 서버에 harness CLI 호출 API 배선(`child_process`).

**공통 필수 작업(무엇을 고르든)**: 세 샘플 다 안 된 **백엔드 연동**. `harness` CLI에 `--json` 출력 옵션을 더하고(`limits`/`mat`/`status.json`), GUI가 이를 주기 호출하면 진짜 라이브 도구가 된다.

**snoopy 처리**: 라우팅 파손·핵심기능 누락·잔재가 많아 토대로 삼기엔 비효율. 참고용으로만.

## 4.5 실제 렌더 검증 (2026-07-04 추가 — 눈으로 확인)

앞 분석은 코드 기반이었으므로, 세 개를 **실제로 실행/렌더해 스크린샷으로 확인**했다. 결과가 코드 예측과 일부 달랐다.

| 항목 | 실행 결과 |
|---|---|
| **claude** | `index.html` 더블클릭 = 즉시 렌더. 코믹 스타일(지그재그 띠·줄무늬 게이지·점 배경) 그대로. **시스템 폰트라 오프라인에서도 동일**. |
| **snoopy** | `npm install` **실패**(vite peer 충돌 → `--legacy-peer-deps` 필요). 실행은 되나 **CDN 폰트 @import가 깨져 코믹 폰트 미적용 → 밋밋한 모노스페이스로 렌더**. 소스배지 없음, calls/tokens/cost 전부 동일 `1,234/45.2K/$2.34` 하드코딩 확인. 코드상 4점짜리 디자인이 **실물에선 구현 안 됨**. |
| **lovable** | **로컬 실행 불가** — export에 `vite.config`가 없어 `vite dev`가 `/`에서 404. .mhtml 스냅샷(온라인 렌더)에서 앱 프레임을 추출해 확인. **가장 완성도 높음**: 둥근 파스텔 패널, 손글씨 폰트(Fraunces+hand) 정상, 이모지 캐릭터 아이콘, 소스배지(추정/실측/원장)·warn 색상·SCORE 칩 모두 표시. 단 폰트가 CDN이라 **온라인 전제**. |

**렌더로 드러난 핵심 교정**:
1. **lovable 디자인 1위는 확인됨** — 실물이 실제로 가장 폴리시 높다.
2. 단 그 폴리시는 **온라인 CDN 폰트에 의존**하고, lovable은 **자체 dev 서버로도 안 돌아간다**(설정 누락). 이식성·실행성은 셋 중 최악.
3. **snoopy의 코믹 디자인은 실물에서 실패**(폰트 미로딩) — 코드만 보고 디자인을 평가하면 틀릴 수 있다는 교훈.
4. **claude만 어디서나(오프라인 포함) 의도대로 렌더**되고, 지그재그 스웨터 모티프는 셋 중 유일.
