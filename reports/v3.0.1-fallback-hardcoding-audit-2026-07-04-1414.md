## 1. 위반 발견 → 수정 (4건)

| # | 위치 | 문제 | 수정 |
|---|---|---|---|
| 1 | `guiserver._run_task_bg` | `except Exception: pass` — GUI 실행 오류를 **조용히 삼킴** | 결과/오류를 `_RUN_STATE["last"]`에 기록, GUI 태스크 탭에 "직전: …" 표시. 검증됨 |
| 2 | `usage._worker_backend_name` | mock 워커 유추 실패 시 조용히 `"claude"` 반환 → **사용량 오귀속** | `None` 반환(지어내지 않음), `today_totals`는 미확정 행 집계 제외. 테스트: unknown-worker 999tok이 claude에 안 붙음 ✅ |
| 3 | `orchestrator._run_verify` | verify 타임아웃 300s 하드코딩 — 테스트 스위트마다 다른 **사용자 환경값** | task 필드 `verify_timeout_sec`(기본 300). 테스트: 1s 설정 → timeout 동작 ✅ |
| 4 | `gui MOCK` | mock 버전 문자열 `"2.3(mock)"` — 실제 3.0과 불일치(스테일 하드코딩) | `"3.0(mock)"` 수정 |

**추가 수정(점검 중 발견한 품질 문제)**
- `usage.today_totals` 원장 전체 재파싱을 매 호출 반복(GUI 7초 폴링 × 호출당 다수) → **mtime/size 기반 캐시** 추가. 데이터를 지어내지 않음(파일 변경 시 즉시 재계산) — 캐시 정합성 테스트 통과(기록 추가 후 값 반영 ✅).
- `orchestrator._repo_context` 주입 예산 6000 하드코딩 → 설정 `repo_context_max_chars`로 이동.
- task `workdir` 오타 시 트레이스백 크래시 → 깔끔한 오류 + aborted 상태 기록. 검증됨.
